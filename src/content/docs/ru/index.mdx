## Установка скилла агента

Дайте Claude Code, Codex, hermes или OpenClaw скилл Maestro одной командой:

```bash
uvx maestro-install skill        # спросит агента; или --agent claude | codex | hermes | openclaw | all
```

Он скачивает и распаковывает скилл в каталог скиллов агента:

- Claude Code → `~/.claude/skills/maestro`
- Codex → `~/.codex/skills/maestro`
- hermes → `~/.hermes/skills/maestro`
- OpenClaw → `~/.openclaw/skills/maestro`

Подробнее: [Agent Skill](care/skill/overview/) · [Установка и использование](care/skill/install/).

## Как это связано

**MAESTRO** — продукт поверх стека из четырёх частей. Каждая часть отвечает за один этап жизни цепочки.

<CardGrid>
	<Card title="MAGE — генерация" icon="seti:python">
		Превращает задачу на естественном языке в CARL-цепочку.
	</Card>
	<Card title="CARL — формат цепочек" icon="open-book">
		Формат, в котором описана каждая цепочка: шаги, зависимости и контекстные запросы.
	</Card>
	<Card title="GigaEvo Memory — хранение" icon="document">
		Хранит цепочки, агентов, навыки и историю запусков.
	</Card>
	<Card title="GigaEvo Platform — эволюция" icon="random">
		Запускает генетический поиск по цепочкам и продвигает лучших.
	</Card>
</CardGrid>

Если коротко: **CARL — это формат, а MAESTRO — система, которая его запускает.** MAGE описывает цепочку в формате CARL, а MAESTRO запускает и эволюционирует её — из терминального интерфейса **MAESTRO CARE** (TUI) или из CLI `care`.
