**MAESTRO** генерирует, запускает и эволюционирует reasoning-цепочки, описанные в
[формате CARL](/ru/carl/getting-started/overview/). Вы управляете системой из
TUI **MAESTRO CARE** (терминальное приложение на Textual) или из headless-CLI
`care`. Это продукт поверх стека из четырёх частей: **MAGE** (генерация),
**CARL** (формат цепочек), **GigaEvo Memory** (хранение) и
**GigaEvo Platform** (эволюция).

## Режимы чата

Переключатель над полем ввода выбирает один из двух режимов (`/mode` переключает
с клавиатуры). По умолчанию — **Ad-Hoc**, настраивается через
`CARE_CHAT__DEFAULT_MODE`.

| Режим | Что происходит на каждый промпт |
| --- | --- |
| **Ad-Hoc** | MAGE генерирует цепочку, MAESTRO сразу её выполняет, ответ выводится инлайн. Агент может зацикливаться (ReAct), пока не решит, что задача выполнена. **Ничего не сохраняется.** |
| **Production** | MAGE генерирует *воспроизводимую* цепочку, MAESTRO сохраняет её в Memory под стабильным `chain_id`, запускает один baseline для затравки датасета и (если подключён Platform) стартует эволюцию. |

Production требует `CARE_MEMORY__BASE_URL`. Без настроенной Memory выбор Production
автоматически откатывается в Ad-Hoc с предупреждением.

## Канонический сценарий

Сгенерировать агента A → сохранить → сгенерировать B и C → вернуться к A из
библиотеки → перезапустить с той же задачей и файлами контекста → опционально
эволюционировать A и принять лучшую особь в стабильный канал.
