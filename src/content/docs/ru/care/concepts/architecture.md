**MAESTRO** — система на вершине стека из четырёх частей; вы работаете с ней через
TUI **MAESTRO CARE** и CLI `care`. Каждая часть отвечает за один этап жизни цепочки.

```text
                ┌──────── GigaEvo Memory ────────┐  (save / load)
                ▼                                 │
You → MAESTRO → MAGE ──CARL-цепочка──▶ MAESTRO ───┘  (запуск)
                                          │
                                          └──evolve──▶ GigaEvo Platform ──winner──▶ Memory
```

| Часть | Пакет | Этап | Роль |
| --- | --- | --- | --- |
| **MAGE** | `mmar-mage` | Генерация | Превращает задачу на естественном языке в CARL-цепочку. |
| **CARL** | `mmar-carl` | Формат цепочек | Формат, в котором описана каждая цепочка — типизированные шаги, зависимости (DAG) и контекст под каждый шаг. См. [документацию CARL](/ru/carl/getting-started/overview/). |
| **GigaEvo Memory** | `gigaevo-client` | Хранение | Хранит сущности (chain / agent / agent_skill / memory_card), библиотеку, историю запусков. |
| **GigaEvo Platform** | — | Эволюция | Генетический поиск по цепочкам; принять и продвинуть победителя. |

## Жизненный цикл

1. **Генерация** — задача описывается словами; MAGE планирует CARL-цепочку.
2. **Запуск** — MAESTRO выполняет цепочку и возвращает результат (с трассировкой токенов и стоимости).
3. **Хранение** — в режиме Production MAESTRO сохраняет цепочку в Memory под
   стабильным `chain_id` и записывает каждый запуск.
4. **Эволюция** — опционально Platform запускает генетический алгоритм над
   цепочкой, а лучшая особь принимается обратно в стабильный канал.

## Канонический сценарий

Сгенерировать агента A → сохранить → сгенерировать B и C → вернуться к A из
[Библиотеки](/ru/care/tui/screens/) → перезапустить с той же задачей и файлами
контекста → опционально эволюционировать A и принять лучшую особь в стабильный канал.

## Плавная деградация

MAESTRO импортирует каждый вышестоящий модуль **лениво** — внутри функции,
которой он нужен. Поэтому минимальная установка по-прежнему запускает CLI и TUI;
отсутствующий компонент (например, опциональный экстра `maestro-care[carl]` или
ненастроенный URL Memory) выдаёт понятную подсказку, а не сбой. Без настроенной
Memory режим Production автоматически откатывается в
[Ad-Hoc](/ru/care/getting-started/overview/).
