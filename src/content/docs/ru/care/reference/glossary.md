| Термин | Значение |
| --- | --- |
| **MAESTRO** | Вся система — генерация, запуск и эволюция reasoning-цепочек. Вы работаете с ней через TUI MAESTRO CARE или CLI `care`. |
| **MAESTRO CARE** | Textual TUI — терминальный интерфейс к MAESTRO (запускается командой `care`). |
| **CARL** | Collaborative Agent Reasoning Library — формат, в котором описываются цепочки рассуждений (и библиотека, которая их исполняет). См. [документацию CARL](/ru/carl/getting-started/overview/). |
| **MAGE** | Генератор, превращающий задачу в CARL-цепочку. |
| **GigaEvo Memory** | Хранилище сохранённых сущностей (цепочки, агенты, навыки, записи запусков) — Библиотека. |
| **GigaEvo Platform** | Сервис эволюции (генетический поиск по цепочкам). |
| **Цепочка (Chain)** | Цепочка рассуждений CARL — единица, которую MAGE генерирует, а CARL исполняет. |
| **`chain_id`** | Стабильный идентификатор, под которым цепочка сохраняется в Memory. |
| **Агент (Agent)** | Сохранённая именованная цепочка в Библиотеке, к которой вы возвращаетесь и повторно запускаете. |
| **AgentSkill** | Переносимая папка навыка (`SKILL.md` + скрипты), которую может запускать цепочка. См. [Возможности](/ru/care/capabilities/overview/). |
| **`memory_card`** | Сохранённая заметка / дайджест запуска в Memory. |
| **Канал (Channel)** | Версионный канал для сохранённой цепочки; `latest` — канал по умолчанию, который читает `care run`. |
| **Родословная (Lineage)** | DAG предков цепочки (родители, от которых она была получена/эволюционировала) — см. `care lineage`. |
| **Приспособленность (Fitness)** | Оценка эволюционировавшей цепочки на датасете; выше — лучше. |
| **Режим Ad-Hoc** | Генерация + запуск инлайн; ничего не сохраняется. |
| **Режим Interactive** | Поверхность чата по умолчанию — генерация цепочки, запуск на месте и ответ инлайн; сохранение / эволюция — на ваше усмотрение (по кнопкам). Устаревшее имя `ad_hoc` сводится к ней. См. [Поверхности чата](/ru/care/workflows/modes/). |
| **Режим Production** | Генерация воспроизводимой цепочки, сохранение в Memory, создание датасета, опциональная эволюция. |
| **Политика стадии** | Как ведёт себя каждая стадия конвейера: `ask` (сначала окно подтверждения), `auto` (выполнить молча) или `skip` (не выполнять и пропустить зависимые). Задаётся для поверхности через `CARE_CHAT__MODE__<MODE>__<STAGE>`. См. [Политику стадий](/ru/care/workflows/modes/). |
| **Мягкий разбор конфига** | Некорректное или устаревшее значение `default_mode` не роняет загрузку конфигурации: известные алиасы нормализуются, всё нераспознанное откатывается к значению по умолчанию (`interactive`) с предупреждением в логе. |
| **Версия (Version)** | Неизменяемая точка в истории сохранённой сущности. Каналы (`latest`, `stable`) — именованные указатели на эту историю — см. `care versions`. |
| **`rollback`** | Переключает канал на конкретную версию — неразрушающая привязка (pin), а не откат (`care rollback`, в TUI — `/rollback`). |
| **`promote`** | Копирует указатель одного канала в другой (по умолчанию `latest → stable`) — `care promote`. В TUI `/promote` добавляет сверху проверку через baseline/оценку. |
| **Бандл библиотеки** | Переносимый tarball с сохранёнными цепочками (+ AgentSkills) для переноса на другую машину — `care export` / `care import`. |
| **Датасет (eval-кейс)** | Набор тест-кейсов цепочки вида `task → expected`, на котором её берут за baseline и оценивают — `care dataset list/add/run/export`. |
| **Долговременная память (LTM)** | Посессионное хранилище заметок, которое сохраняет факты и предпочтения между запусками (`care remember` / `care notes`), отдельно от Библиотеки сущностей. |
| **Agent hub** | Лёгкий локальный процесс, который обслуживает развёрнутую цепочку как HTTP-агента с собственным Swagger UI — `care deploy` / `deployments` / `metrics`. См. [Развёртывание](/ru/care/workflows/deploy/). |
| **Модальное окно DAG** | Полноэкранный вид цепочки (`F`), где рядом стоят граф из блоков и стрелок, список шагов и детали; под `/revise` поверх него рисуется диф новой версии. См. [Чтение DAG](/ru/care/tui/dag/). |
| **Тепловая карта задержек** | Оверлей DAG, затеняющий каждый блок шага по метрике запуска (задержка / токены / стоимость) по шкале от низкого к высокому. |
| **Фронт Парето** | Таблица недоминируемых особей из прогона эволюции; при ≥ 2 целях под ней — 2D-диаграмма рассеяния. |
| **Дашборд эволюции** | Экран TUI (`/evolution`) со списком активных + недавних прогонов; стримит живой график приспособленности, фронт Парето, программы и версии. См. [Эволюцию](/ru/care/tui/evolution/). |
| **Preflight** | Статическая валидация цепочки перед исполнением (`care validate` / `care run`). |
| **Каталог (Catalog)** | Список установленных возможностей (навыки / MCP / инструменты / карточки) — `Ctrl+K` или `care catalog`. |
