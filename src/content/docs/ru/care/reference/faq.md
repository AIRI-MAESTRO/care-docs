## Режим Production постоянно откатывается в Ad-Hoc

Production требует подключения к Memory. Задайте `CARE_MEMORY__BASE_URL` (и
`CARE_MEMORY__API_KEY`, если деплой требует авторизации). Без этого выбор Production
автоматически откатывается в [Ad-Hoc](/ru/care/getting-started/overview/) с
предупреждением. Проверьте с помощью [`care doctor`](/ru/care/cli/setup/).

## «Подсказка по установке» вместо запуска цепочки

Исполнение через CARL — это опциональный экстра `maestro-care[carl]`. Если `care run --execute` (или
запуск в чате) сообщает о недостающей зависимости, установите её:

```bash
uv sync --extra carl     # или: pip install 'maestro-care[carl]'
```

MAESTRO импортирует вышестоящие пакеты лениво, так что минимальная установка всё равно
запустится — подсказка точно укажет, какой экстра добавить.

## Ошибки LLM (401 / 403 / 429)

Они приходят от вашего провайдера и передаются через MAGE/CARL:

- **401** — неверный или отсутствующий API-ключ. Перезапустите [`care init`](/ru/care/cli/setup/) или исправьте `CARE_MAGE__API_KEY`.
- **403** — модель/маршрут не разрешены для вашего ключа. Выберите другую модель.
- **429** — превышен лимит запросов (часто на бесплатных тарифах). Подождите или смените модель/провайдера.

Задайте модель с помощью `CARE_MAGE__MODEL` (например, `qwen/qwen3-coder`).

## Первый запуск открывает Настройки

Это ожидаемое поведение — без `~/.config/care/config.toml` MAESTRO открывает экран **Настроек**,
чтобы вы могли ввести учётные данные MAGE/Memory/Platform. Возвращающиеся пользователи
сразу попадают в чат.

## Мои ключи хранятся в plaintext TOML

Переместите их в keychain ОС с помощью
[`care migrate-secrets`](/ru/care/cli/setup/) (`--dry-run` для предпросмотра).

## Полная проверка состояния

```bash
care doctor          # env + config + отчёт о зависимостях
care doctor --no-probes   # пропустить сетевые проверки (оффлайн / CI)
```
