Сгенерированные цепочки могут выходить за пределы LLM через **инструменты** и **AgentSkills**. MAESTRO
отображает всё установленное через **Каталог** (`Ctrl+K` или
[`care catalog`](/ru/care/cli/discovery/)).

## Инструменты

Инструменты — это Python-функции, которые могут вызывать tool-шаги цепочки.

- **Реестр** — регистрируйте функции с помощью `@carl_tool`; конфигурация `tools` управляет
  реестром (`CARE_TOOLS__*`).
- **Встроенные инструменты** — MAESTRO поставляется с готовыми инструментами (например, `web_search`).
- **Синтез на лету** — когда цепочке нужен незарегистрированный инструмент, MAESTRO
  может синтезировать его из описания, спланировать входные данные и подключить.

См. [tool-шаги CARL](/ru/carl/steps/tool/) о том, как цепочки вызывают инструменты.

## AgentSkills

[AgentSkills](https://agentskills.io) — переносимые папки навыков (`SKILL.md` +
скрипты/ресурсы). Каталог MAESTRO обнаруживает установленные навыки (из `~/.agents/skills/`,
`./.claude/skills/` и библиотеки `agent-skills`), а цепочки запускают их через
шаги AgentSkill. Полная модель — URI, режимы исполнения, резолверы — задокументирована
в [CARL → AgentSkills](/ru/carl/skills/overview/).

## Песочница

Скрипты навыков запускаются внутри **песочницы**, чтобы ненадёжный код оставался изолированным.
Бэкенды: `local`, `docker`, `e2b`, `firejail` (настройка через `CARE_SANDBOX__*`). Доверенные
навыки отслеживаются в хранилище доверия с SHA-пиннингом — его можно проверить и отозвать
на экране **Доверие к песочнице** ([`/sandbox`](/ru/care/slash-commands/overview/)).

## Инжекция контекста

MAESTRO может инжектировать дополнительный контекст при генерации (`CARE_CONTEXT__*`):

- **`CARE.md`** — файл контекста проекта, подхватываемый как системный бриф.
- **Дайджест долгосрочной памяти** — краткое содержание сохранённой памяти, встроенное в промпт.

## Телеметрия

Подключитесь к стриму событий (например, Langfuse) через `CARE_TELEMETRY__*` для трассировки
генерации + исполнения в дашборде.
