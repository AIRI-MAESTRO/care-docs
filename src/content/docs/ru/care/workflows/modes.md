Переключатель над промптом (или `/mode`) задаёт, как MAESTRO обрабатывает каждое
сообщение. Поверхностей три, но настоящих режима всего два — **Interactive** (по
умолчанию) и **Production**. **Ad-Hoc** — устаревшее название, которое при чтении
сводится к Interactive.

| | **Interactive** (по умолчанию) | **Production** |
| --- | --- | --- |
| На каждом промпте | Сгенерировать цепочку, выполнить её **на месте**, ответить inline. | Сгенерировать **воспроизводимую** цепочку, затем автоматически save → baseline → (опционально) evolve. |
| Save / baseline / evolve | **На ваше усмотрение** — управляется кнопками действий над цепочкой (Save to library, Evolve). | **Автоматически** — конвейер выполняет их за вас. |
| Следующий промпт | **Переиспользует цепочку** как контекст; новый промпт может сгенерировать заново. | Трактуется как [`/revise`](/ru/care/slash-commands/overview/) — новая версия той же цепочки. |
| Сохранение? | Только когда вы нажмёте **Save**. | Сохраняется в Memory под стабильным `chain_id` + запись baseline-датасета. |
| Нужна Memory? | Нет. | **Да** (`CARE_MEMORY__BASE_URL`). |

## Interactive

Поверхность по умолчанию. Введите задачу → MAGE генерирует → CARL выполняет →
печатается ответ. Ничего не сохраняется, пока вы сами этого не захотите: кнопки
действий **Save to library** и **Evolve** оставляют за вами решение, что
переживёт сессию. Последующие промпты переиспользуют ту же цепочку, чтобы у
разговора был контекст; начать новую цепочку можно в любой момент.

Используйте для: быстрых ответов, исследования, доводки цепочки до того, как её
зафиксировать.

## Production

Долговечный путь — для агентов, которые вы будете хранить, измерять и улучшать.
Каждый промпт создаёт одну **воспроизводимую** цепочку, которая затем
**автоматически** сохраняется, прогоняется как baseline и (если подключён
Platform) эволюционирует. Как только цепочка существует, следующие обычные
сообщения трактуются как правки `/revise` — новая версия под тем же `chain_id`, а
не новая цепочка. Детали и полный жизненный цикл — на странице
[Режим Production](/ru/care/workflows/production/).

Используйте для: агентов, которых вы хотите сохранить, для которых хотите собрать
датасет, эволюционировать и продвигать.

:::note
Production требует `CARE_MEMORY__BASE_URL` (и `CARE_MEMORY__API_KEY`, если
развёртывание требует аутентификации). Задайте стартовую поверхность через
`CARE_CHAT__DEFAULT_MODE`.
:::

## Ad-Hoc (устаревшее)

`ad_hoc` (а также написания `ad-hoc` / `adhoc`) — старое название быстрого пути
с выполнением на месте. Оно сохранено только ради обратной совместимости: при
чтении значение **сводится к `interactive`**, поэтому всё, что по-прежнему
передаёт `ad_hoc`, попадает на поверхность Interactive без изменений в поведении.

## Политика стадий

Обе поверхности собраны из одного конвейера. Две стадии — **generate** и
**preview** — всегда автоматические. Остальные четыре настраиваемые, у каждой
одна из трёх политик:

| Политика | Поведение |
| --- | --- |
| `auto` | Выполнить молча. |
| `ask` | Сначала показать окно подтверждения. |
| `skip` | Не выполнять вовсе (и пропустить стадии, которые от неё зависят). |

Четыре настраиваемые стадии и их пресеты по поверхностям:

| Стадия | Interactive | Production |
| --- | --- | --- |
| `run` | `ask` | `ask` |
| `save` | `skip` (по кнопке) | `auto` |
| `baseline` | `skip` (по кнопке) | `auto` |
| `evolve` | `skip` (по кнопке) | `auto` |

В Interactive стадии `save` / `baseline` / `evolve` имеют политику `skip` на
уровне конвейера, потому что управляются **кнопками** действий над цепочкой, а не
модальными окнами.

## Env-переопределения

Задайте стартовую поверхность, а затем переопределите политику любой стадии, не
правя `config.toml`. Env-переменные вкладываются через разделитель `__` на каждом
уровне.

```bash
# Стартовая поверхность: interactive | production (устаревшие алиасы принимаются)
export CARE_CHAT__DEFAULT_MODE=production

# Политика стадии: CARE_CHAT__MODE__<MODE>__<STAGE> = ask | auto | skip
export CARE_CHAT__MODE__INTERACTIVE__RUN=auto      # запускать Interactive-цепочки без подтверждения
export CARE_CHAT__MODE__PRODUCTION__SAVE=ask       # подтверждать сохранение в Production
```

`<MODE>` — это `INTERACTIVE` или `PRODUCTION`; `<STAGE>` — `RUN`, `SAVE`,
`BASELINE` или `EVOLVE`. Незаданная стадия наследует пресет выше.

:::note[Мягкий разбор]
Некорректное или устаревшее значение `default_mode` никогда не роняет загрузку
конфигурации. Известные алиасы (`ad_hoc`, `ad-hoc`, `adhoc` → `interactive`;
`prod` → `production`) нормализуются; всё нераспознанное отбрасывается, поле
откатывается к значению по умолчанию (`interactive`), и в лог пишется
предупреждение.
:::
