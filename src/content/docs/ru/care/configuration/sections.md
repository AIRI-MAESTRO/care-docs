Ключевые параметры по каждой секции. Каждый ключ также является env-переменной
`CARE_<СЕКЦИЯ>__<КЛЮЧ>`. Файл
[`.env.example`](https://github.com/Glazkoff/care/blob/main/.env.example) в репозитории —
полный аннотированный список всех параметров.

## `mage` — генератор

Генератор MAGE, превращающий задачу в цепочку. Минимум, необходимый новому
чекауту (записывается [`care init`](/ru/care/cli/setup/)).

| Ключ | Описание |
| --- | --- |
| `CARE_MAGE__API_KEY` | API-ключ LLM. |
| `CARE_MAGE__BASE_URL` | Базовый URL, совместимый с OpenAI (например, OpenRouter). |
| `CARE_MAGE__MODEL` | Идентификатор модели (например, `qwen/qwen3-coder`). |
| `CARE_MAGE__MODE` | Режим генерации. |

## `memory` и `platform`

| Ключ | Описание |
| --- | --- |
| `CARE_MEMORY__BASE_URL` | URL GigaEvo Memory — **обязателен для режима Production**. |
| `CARE_MEMORY__API_KEY` | Если деплой требует авторизации. |
| `CARE_PLATFORM__BASE_URL` | URL GigaEvo Platform (запуски эволюции). |

## `chat` — интерфейс чата

| Ключ | По умолчанию | Описание |
| --- | --- | --- |
| `CARE_CHAT__DEFAULT_MODE` | `interactive` | [Режим](/ru/care/getting-started/overview/) при запуске — `interactive` или `production`. Устаревшие алиасы `ad_hoc` / `ad-hoc` / `adhoc` (и `prod`) при чтении приводятся к каноническому идентификатору. |
| `CARE_CHAT__AD_HOC_HISTORY_TURNS` | `6` | Окно контекста для уточняющих вопросов (ходов). |
| `CARE_CHAT__AD_HOC_HISTORY_CHARS` | `1200` | Символов на запомненный ход. |
| `CARE_CHAT__GENERATION_MAX_ATTEMPTS` | `3` | Повторных попыток генерации MAGE. |

### Переопределение стадий по режимам

В каждом режиме конвейер проходит четыре настраиваемые стадии — **run**, **save**,
**baseline**, **evolve**, — и у каждой есть пресет, свой для режима. Любую отдельную
стадию можно переопределить env-переменной по шаблону `CARE_CHAT__MODE__<РЕЖИМ>__<СТАДИЯ>`,
где `<РЕЖИМ>` — это `INTERACTIVE` или `PRODUCTION`, а `<СТАДИЯ>` — `RUN` / `SAVE` /
`BASELINE` / `EVOLVE`. Значение задаёт политику стадии:

| Значение | Поведение |
| --- | --- |
| `ask` | Сначала спросить подтверждение перед стадией. |
| `auto` | Выполнить стадию молча. |
| `skip` | Не выполнять стадию (и пропустить зависящие от неё). |

```bash
# В Interactive запускать автоматически, а Save в Production — с подтверждением
CARE_CHAT__MODE__INTERACTIVE__RUN=auto
CARE_CHAT__MODE__PRODUCTION__SAVE=ask
```

Незаданная стадия наследует пресет режима.

## Остальное

| Секция | Что настраивать |
| --- | --- |
| `hub` | `CARE_HUB__*` — [agent hub](/ru/care/workflows/deploy/) для `/deploy` (`base_url`, `port`, `autostart`, `state_file`). |
| `upload` | `CARE_UPLOAD__URL` — эндпоинт для `/upload`. |
| `sandbox` | Бэкенд (`local` / `docker` / `e2b` / `firejail`) + лимиты ресурсов для AgentSkills. |
| `tools` | Реестр `@carl_tool`, встроенные инструменты (web_search…), синтез на лету. |
| `telemetry` | Opt-in стрим событий (например, публичный/секретный ключи Langfuse). |
| `defaults` | Значения UI по умолчанию — язык, размер истории, [отрисовка DAG](#отрисовка-dag-по-умолчанию). |
| `context` | Инжекция пользовательского контекста (CARE.md + дайджест долгосрочной памяти). |
| `artifacts` | Директория хранения сохранённых артефактов. |

### Отрисовка DAG по умолчанию

Как `[defaults]` рисует DAG цепочек на всех поверхностях (лента чата, панель
просмотра, оверлей запуска, модальное окно DAG).

| Ключ | По умолчанию | Описание |
| --- | --- | --- |
| `CARE_DEFAULTS__DAG_LAYOUT` | `tb` | Начальная ориентация DAG в модальном окне — `tb` (сверху вниз) или `lr` (слева направо). Переключается на лету клавишей `l`. |
| `CARE_DEFAULTS__DAG_ASCII` | `false` | Рисовать DAG обычными ASCII-символами вместо Unicode-псевдографики — для терминалов и шрифтов, не умеющих рендерить рамочные глифы, или для чистого копипаста. |
| `CARE_DEFAULTS__DAG_BUS_LANES` | `false` | Прокладывать межслойные зависимости-«пропуски» отдельными каналами по левому краю вместо лаконичных пометок `◀ N` на зависимом блоке. |

## Секреты

Держите API-ключи подальше от plaintext TOML с помощью
[`care migrate-secrets`](/ru/care/cli/setup/) — команда перемещает их в keychain ОС.
