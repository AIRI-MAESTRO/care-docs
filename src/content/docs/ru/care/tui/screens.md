MAESTRO CARE поставляется с экраном для каждого этапа жизненного цикла. Три наиболее часто
используемых — чат, **Библиотека** и экран **Эволюции**.

## Основные экраны

| Экран | Как открыть | Назначение |
| --- | --- | --- |
| Чат | запуск | Ввод на естественном языке, переключатель режима, палитра слэшей, таблетка артефактов, ссылка на Библиотеку в шапке, панель Production. |
| DAG цепочки | Чат → **Read full** | Полный граф «прямоугольники и стрелки» свежесгенерированной цепочки + кликабельный список шагов + панель деталей CARL (см. [DAG цепочки](/ru/care/tui/dag/)). |
| Артефакты | `/artifacts` | Артефакты текущего чата (chain / stage / tool / dataset / synth output); сохранить, скопировать, просмотреть, переключатель JSON/DAG (см. [ниже](#экран-артефактов)). |
| Библиотека | `/library` · ссылка в шапке | Сохранённые цепочки — сортируемые колонки, фильтр, пул тегов, полоса актуальности, средняя стоимость, массовый импорт/экспорт (см. [ниже](#экран-библиотеки)). |
| Инспекция | `Enter` в Библиотеке | Детали сохранённой цепочки + история запусков + панель интеграции. |
| Редактировать агента | `e` в Библиотеке | Ручное редактирование — имя / описание / теги / задача / сводка изменений, сохранить как новую версию или продвинуть в стабильный (см. [ниже](#экран-редактирования-агента)). |
| Исполнение | `r` в Библиотеке | Живой запуск CARL + потоковая передача токенов. |
| Эволюция | `/evolution` · `v`/`E` в Библиотеке | Запуск + наблюдение за ГА над цепочкой (см. [Экран эволюции](/ru/care/tui/evolution/)). |
| Дашборд эволюции | `/evolution` | Список активных + последних запусков; Enter открывает запуск, `c` сравнивает два. |
| Воспроизведение | `Enter` в Запусках | Пройти по сохранённому `ReasoningResult`. |
| Запуски | `/runs` | Локальная история запусков; Enter открывает сайдкар Воспроизведения. |
| Каталог | `Ctrl+K` | Просмотр установленных возможностей (навыки / MCP / инструменты / карточки). |
| Дашборд стоимости | `/cost` | Сводка расходов токенов + USD по провайдеру / цепочке / сессии. |
| Маркетплейс | `/marketplace` | Поиск общих листингов `agent_skill` в Memory. |
| Логи | `/logs` | Хвост скользящего лога приложения; `m` переключает фильтр модуля. |
| Доверие к песочнице | `/sandbox` | Аудит + отзыв доверенных AgentSkills (хранилище доверия с SHA-пиннингом). |
| Профиль | `/profile` | Список профилей учётных данных в `~/.config/care/profiles/`. |
| Настройки | `/settings` | Редактирование учётных данных MAGE / Memory / Platform + тема + продвинутые параметры. |
| Справка | `/help` · `?` | Туториал + все привязки (отфильтрованные по активному экрану). |
| Приветствие | запуск | Заставка при загрузке; ведёт в Чат (возвращающихся) или Настройки (первый запуск). |
| Список задач | `Ctrl+B` | Панель работающих воркеров. |

## Модальные окна

| Модальное окно | Открывается из | Назначение |
| --- | --- | --- |
| Палитра команд | `Ctrl+P` | Fuzzy-палитра по командам + сохранённым сущностям. |
| Подтверждение | деструктивные действия | OK / Отмена для массового удаления, принятия победителя, … |
| Конфликт | коллизия имён при сохранении | Разрешить коллизию имён при сохранении. |
| Diff | `D` в Библиотеке | Сравнение двух цепочек / особи с родителем бок о бок. |
| Родословная | `l` в Библиотеке | Прогулка по DAG предков цепочки. |
| Импорт / Экспорт | `i` / `x` в Библиотеке | Импорт бандла цепочки / экспорт сущностей в tarball. |
| Экспорт цепочки | `x` в Эволюции | Экспорт одной цепочки на диск — выбор **Markdown / JSON / Python** с пикером папки **Browse**. |
| Запуск эволюции | `v` / `E` в Библиотеке | Выбор бюджета / рубрики / целей перед эволюцией. |
| Сравнение эволюций | `c` на Дашборде | Кривые приспособленности двух запусков бок о бок. |
| Ввод человека | шаг human-input в CARL | Остановка исполнения для получения ответа от человека. |
| Восстановление | `/resume` | Восстановить транскрипт режима Production. |
| Контекст запуска | `r` в Библиотеке / Исполнении | Форма перезапуска: задача + выбор файлов контекста + теги. |
| Сохранение агента | после генерации | Теги + имя для только что сгенерированной цепочки. |
| Отчёт о сохранении | после save-all | Таблица постмортема результатов save-all. |
| Редактор тегов | `T` в Библиотеке | Редактирование тегов (массовое) + опциональный заголовок. |
| Использовать сейчас | после сохранения | Рецепт copy-paste (python / curl / cli) для сохранённой цепочки. |

## Экран библиотеки

`/library` — или ссылка **Библиотека** в шапке Чата — показывает все сохранённые цепочки.
Клик по заголовку колонки сортирует список: сортируются **Name**, **Last Run** и **Runs**
(число запусков); сортировка сохраняется между визитами, а избранные всегда закреплены
сверху. В каждой строке есть **полоса актуальности** (последний запуск + бейдж
успешность/N) и колонка **средней стоимости** из локальной истории запусков. `e`
редактирует строку, `r` перезапускает её, `Enter` открывает инспекцию; `i`/`x` импортируют
или экспортируют бандлы.

## Экран редактирования агента

`e` в Библиотеке открывает экран **ручного редактирования** сохранённой цепочки. Можно
изменить **имя**, **описание**, **теги**, **описание задачи** и **сводку изменений**; во
вкладке Content также доступно ручное редактирование CARL JSON. Валидация срабатывает на
каждое нажатие. Внизу — **Save** (`Ctrl+S` — записывает **новую версию**), **Promote**
(продвигает цепочку в канал **stable**) и **Back** (`Esc`). Кнопка Save показывает, сколько
полей изменилось, а при структурном правке сводка изменений подставляется автоматически,
чтобы сохранение не блокировалось молча.

## Экран артефактов

`/artifacts` — двухпанельный браузер по всему, что произвела текущая сессия чата (цепочки,
полезные нагрузки шагов, вывод инструментов, датасеты, синтезированные ответы). В шапке —
**таблетка артефактов** `[ N · M unsaved ]`, синхронизированная с таблеткой на экране чата:
сохранение цепочки (`s`) уменьшает счётчик несохранённых, удаление одной (`d`) — общий
счётчик. `v` переключает панель деталей между сырым **JSON** и графом шагов **DAG**; `Enter`
открывает инспекцию цепочки, `c` копирует её полезную нагрузку, `S` сохраняет все
несохранённые, `D` сравнивает две выбранные строки.
