`/evolution` открывает **панель эволюции** MAESTRO CARE — список всех прогонов
генетического алгоритма, которые видны вашей учётной записи на Platform. Из строки
вы открываете **экран прогона** с живой трансляцией поиска: кривые приспособленности,
фронт Парето, динамика валидности программ и лестница роста приспособленности по
поколениям. Когда прогон находит победителя, вы принимаете его обратно в канал
**stable**.

Это визуальный аналог [`care evolve`](/ru/care/cli/overview/). Панель наблюдает за
прогонами и сравнивает их; CLI запускает их в headless-режиме. Про сам алгоритм —
мутации, оценку, поколения — см. концептуальную страницу
[Обзор эволюции](/ru/carl/evolution/overview/).

## Панель

Панель — это единая сортируемая таблица, где сверху самые свежие прогоны. Две
вкладки вверху делят её на **Active** (по умолчанию — живые и недавние прогоны) и
**Archived** (прогоны, которые вы скрыли, но не удалили).

| Столбец | Что показывает |
| --- | --- |
| **Status** | Цветной кружок `●` + статус прогона: зелёный — выполняется, жёлтый — в очереди/на паузе, синий — завершён, серый — остановлен, красный — сбой. |
| **Run** | Идентификатор прогона на Platform. |
| **Base chain** | Исходная цепочка, от которой эволюционирует прогон. |
| **Gen** | Текущее поколение. |
| **Best** | Лучшая приспособленность за все поколения. |
| **Current** | Лучшая приспособленность в текущем поколении. |
| **Valid** / **Invalid** | Количество валидных и невалидных программ. |
| **Started** | Локальное время старта прогона. |
| **Elapsed** | Время с момента старта; отсчёт останавливается, когда прогон завершается. |

Неизвестные ячейки отображаются как `—`, поэтому неполный ответ Platform никогда не
обнулит целую строку. Таблица автоматически обновляется каждые 5 секунд, так что
долгое наблюдение остаётся актуальным.

### Клавиши

| Клавиша | Действие |
| --- | --- |
| `Enter` | Открыть выделенный прогон (см. [Экран прогона](#экран-прогона)). |
| `s` | Остановить выделенный работающий прогон. |
| `Space` | Отметить строку для сравнения (не больше двух). |
| `c` | Сравнить два отмеченных прогона. |
| `a` | Архивировать / разархивировать выделенный прогон. |
| `r` | Обновить сейчас. |
| `b` · `Esc` | Назад в чат. |

Те же действия доступны кнопками внизу — для тех, кто ещё не запомнил привязки.

:::note[Остановка работает «по возможности»]
`s` просит Platform отменить прогон. Если настроенный SDK не предоставляет метод
отмены, панель сообщит об этом всплывающим уведомлением, а не упадёт молча.
:::

### Счётчик «▶ N evolving»

Пока открыт интерфейс чата, в футере есть живой сегмент **`▶ N evolving`**,
считающий ваши работающие прогоны. Он опрашивает Platform вне UI-потока и прячется,
когда ничего не выполняется, — тихий сигнал «одним взглядом», что работа всё ещё
идёт в фоне.

### Сравнение двух прогонов

Отметьте две строки клавишей `Space`, затем нажмите `c`, чтобы открыть модальное
окно **Evolution Compare**. Первая отмеченная строка — слева, вторая — справа, и
окно рисует обе кривые приспособленности рядом, чтобы видеть, какой прогон рос
быстрее или раньше вышел на плато.

### Архивация

Отменённые и сбойные прогоны накапливаются. Нажмите `a`, чтобы перенести строку из
вкладки **Active** в **Archived** — прогон останется на Platform, а список архива
сохранится в `~/.config/care/evolution_archive.json` и переживёт перезапуски.
Переключитесь на вкладку **Archived** и снова нажмите `a` на строке, чтобы вернуть
её обратно.

## Экран прогона

`Enter` на строке панели — или **Библиотека → `v`/`E`**, пикер
[Запуска эволюции](/ru/care/tui/screens/) — открывает экран прогона. Открытый с
панели, он наблюдает за уже существующим прогоном (возобновляет живую трансляцию,
ничего не отправляя заново); запущенный из пикера, он стартует новый прогон и
следит за ним.

Сверху идут панель **Status** и лог **Events**; остальное — вкладочный блок,
повторяющий веб-интерфейс gigaevo-platform, чтобы вы видели одну и ту же подачу
информации на любой из поверхностей.

| Вкладка | Что показывает |
| --- | --- |
| **Fitness** | Двухосевой линейный график высокого разрешения — **лучшая приспособленность** и **среднее по текущему поколению** относительно поколения — перерисовывается по мере поступления точек, чтобы следить за прогрессом и замечать плато. |
| **Statistics** | Сводные карточки плюс карточка **«что оптимизируется»** (см. ниже). |
| **Pareto front** | Таблица недоминируемых особей; выделенная строка раскрывается в карточку с полным резюме и всеми целями. Прогоны с ≥ 2 целями получают под таблицей ещё и **2D-диаграмму рассеяния**. |
| **Programs** | Динамика валидных и невалидных программ по поколениям. |
| **Versions** | Лестница роста приспособленности — каждое поколение, где лучшая приспособленность строго выросла, строками `Gen | Fitness | Δ`. Выберите одну, чтобы предпросмотреть её цепочку или сравнить с родителем. |

### Что оптимизируется

Вкладка **Statistics** отвечает на вопрос «что этот прогон вообще измеряет?»
карточкой метаданных, собранной из:

- **Optimising for** — рубрика валидации (из конфигурации запуска или восстановленная
  из описания прогона, когда вы наблюдаете за чужим прогоном);
- **Mode** — `full_chain` (весь DAG в игре) или `per_step` (за раз изменяется один шаг);
- **Runner** — раннер Platform, выполнивший прогон;
- **Platform** — версия Platform, запрашиваемая один раз при монтировании.

### Счётчик стоимости и токенов

Экран по ходу прогона собирает расход токенов и USD прямо из живой трансляции — из
отдельных событий `cost_tick` либо из полей токенов/usage, едущих в событиях по
отдельным особям, когда Platform не присылает отдельные тики. Текущая сумма
токенов (prompt + completion) и накопленная стоимость в USD отображаются в области
статуса, чтобы видеть расходы поиска в реальном времени.

### Живые обновления с резервным опросом

Экран прогона подписывается на поток событий SSE от Platform и отрисовывает каждое
событие `generation_started`, `individual_evaluated` и `best_updated` по мере
поступления. Когда поток недоступен, он переходит на опрос метрик `/results`
прогона с тиком ~2 секунды (а для локального стека с пустым `/results` — на прямой
зонд Redis для кривой), так что вид продолжает двигаться даже без живого
SSE-соединения.

### Предпросмотр бюджета запуска

Пикер [Запуска эволюции](/ru/care/tui/screens/) показывает живой **предпросмотр
бюджета** перед отправкой. Он пересчитывается по мере правки полей:

```text
evaluations = iterations × population
```

плюс грубую **оценку токенов** на такое число оценок. Тогда прикинуть размер
прогона — дело одного взгляда: подкрутите итерации или размер популяции и смотрите,
как меняется прогноз расходов, прежде чем подтвердить.

### Принять победителя и экспорт

| Клавиша | Действие |
| --- | --- |
| `a` | **Accept winner** — продвинуть выбранную особь (или лучшую) в канал **stable** после подтверждения, где показаны id цепочки и переход версии. |
| `x` | Экспортировать цепочку выделенной особи с фронта Парето на диск (переключатель JSON / Python). |
| `c` | Экспортировать кривую приспособленности в `evolution-<id>-curve.csv` + `.json` в текущий каталог. |
| `D` | Сравнить выбранную версию с родителем. |
| `z` | **Stop + Archive** — остановить прогон на стороне сервера и архивировать его. |
| `Esc` | Отмена. |

Принятие — единственный необратимый шаг (он переставляет указатель `latest`),
поэтому всегда проходит через модальное окно подтверждения. Если выбрать на фронте
Парето не лучшую строку, принятие продвинет именно цепочку *этой* особи, а не общую
лучшую.
