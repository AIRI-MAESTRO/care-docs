CARL 0.3.0 добавил типизированный API, который [MAESTRO](/ru/care/getting-started/overview/)
(и любой инструмент с поддержкой MAESTRO) использует для сохранения, перезапуска и интроспекции цепочек.
Вы тоже можете использовать его напрямую.

| Компонент | Для чего |
| --- | --- |
| [Preflight](/ru/carl/care-integration/preflight/) | Статически узнать, что нужно цепочке (инструменты / MCP-серверы / навыки) до запуска — и чего не хватает. |
| [RunRecord](/ru/carl/care-integration/run-record/) | Долговечная, воспроизводимая аудит-запись одного выполнения (цепочка + вход + результат + тайминги). |
| [Метаданные MAESTRO](/ru/carl/care-integration/metadata/) | Типизированный блок происхождения на цепочке (задача, прикреплённые файлы, теги) — и повторная инициализация контекста из него. |

Это строительные блоки за Библиотекой MAESTRO: сохранить сгенерированную цепочку с её
метаданными, выполнить preflight перед перезапуском, выполнить и сохранить `RunRecord`
каждого прогона.
