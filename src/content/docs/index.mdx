## Install the agent skill

Give Claude Code, Codex, hermes, or OpenClaw the Maestro skill in one command:

```bash
uvx maestro-install skill        # asks which agent; or --agent claude | codex | hermes | openclaw | all
```

It downloads and unpacks the skill into the agent's skills directory:

- Claude Code → `~/.claude/skills/maestro`
- Codex → `~/.codex/skills/maestro`
- hermes → `~/.hermes/skills/maestro`
- OpenClaw → `~/.openclaw/skills/maestro`

More: [Agent Skill](care/skill/overview/) · [Install & Use](care/skill/install/).

## How it fits together

**MAESTRO** is the product on top of a four-part stack. Each part owns one stage of a chain's life.

<CardGrid>
	<Card title="MAGE — generation" icon="seti:python">
		Turns a natural-language task into a CARL chain.
	</Card>
	<Card title="CARL — the chain format" icon="open-book">
		The format every chain is written in: steps, dependencies, and context queries.
	</Card>
	<Card title="GigaEvo Memory — persistence" icon="document">
		Stores chains, agents, skills, and run history.
	</Card>
	<Card title="GigaEvo Platform — evolution" icon="random">
		Runs a genetic search over chains and promotes the winners.
	</Card>
</CardGrid>

In short: **CARL is the format, MAESTRO is the system that runs it.** MAGE writes a chain in the CARL format; MAESTRO runs and evolves it — from the **MAESTRO CARE** TUI or the `care` CLI.
